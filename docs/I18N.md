# 多言語対応（i18n）設計

**ステータス:** 計画（実装未着手）
**出典:** 旧 `README_plan.md` §11。**実装方式を独自フックから i18next / react-i18next（Primadoc 同等）へ変更。**

---

## 1. 基本方針
- 最初から多言語対応を組み込み、後々の実装を容易にする
- 日本語・英語の 2 言語から開始（将来 2〜4 言語を想定）
- **i18next + react-i18next** を採用（Primadoc と統一）。文字列はハードコードせず必ず翻訳リソースで管理する。

> 旧計画では「1 ファイル内に `{ja, en}` を併記する独自方式 + 自作 `useTranslation`」だったが、Primadoc 同等スタックに合わせ i18next に統一する。言語ごとにリソースを分けることで、`react-i18next` の `useTranslation`・補間・複数形・名前空間をそのまま利用できる。

---

## 2. リソース構造

言語別 × 名前空間（namespace）でファイルを分割する。

```
src/renderer/locales/
├── ja/
│   ├── common.json     # 共通（ボタン・ラベル・メニュー）
│   ├── mail.json       # メール関連
│   ├── settings.json   # 設定
│   ├── search.json     # 検索
│   ├── tags.json       # タグ
│   └── errors.json     # エラーメッセージ
└── en/
    ├── common.json
    ├── mail.json
    ├── settings.json
    ├── search.json
    ├── tags.json
    └── errors.json
```

## 3. セットアップ

```typescript
// src/renderer/i18n.ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';

import jaCommon from './locales/ja/common.json';
import jaMail from './locales/ja/mail.json';
import enCommon from './locales/en/common.json';
import enMail from './locales/en/mail.json';
// ... 他の名前空間も同様に import

export const SUPPORTED_LANGUAGES = ['ja', 'en'] as const;
export type SupportedLanguage = (typeof SUPPORTED_LANGUAGES)[number];

i18n.use(initReactI18next).init({
  resources: {
    ja: { common: jaCommon, mail: jaMail /* ... */ },
    en: { common: enCommon, mail: enMail /* ... */ },
  },
  lng: 'ja',
  fallbackLng: 'en',
  ns: ['common', 'mail', 'settings', 'search', 'tags', 'errors'],
  defaultNS: 'common',
  interpolation: { escapeValue: false },
});

export default i18n;
```

## 4. コンポーネントでの使用

```tsx
// src/renderer/components/mail/MailList.tsx
import { useTranslation } from 'react-i18next';

export const MailList: React.FC = () => {
  const { t } = useTranslation('mail');

  return (
    <div>
      <h1>{t('mailList.title')}</h1>
      <span>{t('mailList.unread')}: 5</span>
      <p>{t('mailList.mailCount', { count: 10 })}</p>
      {emails.length === 0 && <p>{t('mailList.noMails')}</p>}
    </div>
  );
};
```

言語切替:

```tsx
import { useTranslation } from 'react-i18next';
const { i18n } = useTranslation();
i18n.changeLanguage('en');
```

## 5. 翻訳の整合性チェック

言語間でキーの欠落がないかを検証する。Primadoc の運用に合わせ、CI / lint-staged に組み込む。

- ja と en で**キー集合が一致**しているかを比較
- 欠落キー・余剰キーを検出してエラー表示
- 補間プレースホルダ（`{{count}}` 等）の整合も確認

> 旧計画の `scripts/validate-translations.js` は「1 ファイル内 `{ja,en}`」前提だったため、i18next の言語別ファイル構成に合わせた検証スクリプトへ作り替える。

---

## 6. メリット
1. **エコシステム**: 複数形・補間・名前空間・遅延読込など i18next の機能をそのまま利用
2. **Primadoc と統一**: 規約・ツールチェーンを共有でき学習コストが低い
3. **保守性**: 言語別ファイルで翻訳作業を分担しやすい
4. **拡張性**: 新言語の追加はディレクトリ追加 + リソース登録のみ
