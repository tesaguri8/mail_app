# メールの安全表示（リモート画像ブロック・なりすまし/危険警告）

**ステータス:** 計画（実装未着手）／プライバシー看板に直結
**目的:** 受信メールを**安全に・追跡されずに**読む。リモート画像（追跡ピクセル）の既定ブロックと、なりすまし/フィッシングの警告表示。

関連: [POSITIONING.md](POSITIONING.md)（プライバシー看板）/ [THREADING.md](THREADING.md)（ヘッダ活用）/ [SPAM.md](SPAM.md) / [FILTERING.md](FILTERING.md)

---

## 1. リモート画像/外部コンテンツの既定ブロック

HTMLメールの外部画像は**開封トラッキング（追跡ピクセル）**になり得る。

- **既定でブロック**: 外部画像・CSS・フォント等を読み込まない。プレースホルダ＋「画像を表示」ボタンを表示。
- **許可の粒度**: この差出人は常に許可 / この1通だけ許可 / 全許可（非推奨）。許可は `contacts.allow_remote_images` 等で保持。
- **追跡ピクセル検出**: 1x1・既知トラッカードメイン等を検出して可視化（「このメールに追跡画像 N 個」）。
- **（任意・将来）プロキシ取得**: 画像を TSG One プロキシ経由で取得し IP を隠す。サーバー依存になるためオプション。
- **インライン画像（cid:）はブロック対象外**（添付に含まれるため追跡にならない）。

> 看板「AIにもトラッカーにも勝手に渡さない」に直結。実装も比較的軽い。**最初の差別化として優先**。

## 2. なりすまし/フィッシング警告

差出人の正当性と危険サインを**画面で分かりやすく**示す。

### 認証結果（[THREADING.md](THREADING.md) §7.5 の `Authentication-Results` を活用）
- **SPF / DKIM / DMARC** の pass/fail をバッジ表示。fail や不一致は警告色。
- `From` と `Return-Path`/`Reply-To` の不一致を提示。

### なりすまし検知
- **表示名スプーフィング**: 既知の表示名なのに**アドレスが普段と違う**（連絡先と照合）→ 警告。
- **類似ドメイン（ホモグラフ）**: punycode/紛らわしい綴り（`rn`↔`m` 等）を検出。
- **初回の差出人**: 「初めての相手」インジケータ。

### 危険リンク/添付
- **リンク偽装**: 表示テキストと実 URL（href）の不一致、punycode、既知の悪性ドメインを警告。クリック前に実リンクをプレビュー。
- **危険な添付**: 実行ファイル・二重拡張子（`.pdf.exe`）・マクロ文書を警告。暗号化アーカイブは「中身を検査できない」注記。

> 素材は標準ヘッダ＋連絡先照合。**表示設計（バッジ・警告の見せ方）**を本書で定義し、過剰警告にならない粒度にする。

## 3. データモデル（要約）

- `contacts.allow_remote_images`（差出人ごとの画像許可）
- `emails.auth_result`（既存。SPF/DKIM/DMARC サマリ）を表示に利用
- ブロック状態・追跡検出数は表示時に算出（必要なら `emails` にキャッシュ列）

## 4. Tauri コマンド（抜粋）

| コマンド | 用途 |
|---|---|
| `mail_render_safe` | リモート画像を抑止して本文をレンダリング |
| `mail_load_remote` | 明示的に外部画像を読み込む（この1通/この差出人） |
| `sender_set_remote_policy` | 差出人ごとの画像許可を設定 |
| `mail_security_report` | 認証結果・なりすまし/危険サインの一覧を返す |
