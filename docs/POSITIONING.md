# プロダクト・ポジショニング / MVP スコープ（北極星）

**ステータス:** 計画（実装未着手）
**役割:** 機能追加で迷ったときに立ち返る“北極星”。看板メッセージ・差別化・尖った MVP の線引きを定める。

---

## 1. 看板メッセージ（North Star）

> **Windows で唯一の、軽くて美しくて、AIに勝手に中身を渡さない、所有したくなるメール。**

- 普段は**美しい背景＋時計・日付のウィジェット**として置いておきたくなる。
- 開けば**チャット形式**で会話が流れ、長い引用や「同件名・別内容」に悩まされない。
- **AIに不用意にプライバシーを渡さない**（保護領域・伏字・ローカル処理）。

---

## 2. 誰のため / どの不満を解くか

- **対象**: 美しさ・心地よさ・軽快さ・プライバシーを重視する Windows ユーザー（まずは作者自身）。
- **市場の穴**: Windows には「**無料・軽量・美しい・所有したくなる**」メールクライアントが事実上無い。
  - 新 Outlook は Web ラッパー化で不評／Thunderbird は質実剛健で“所有欲”を満たさない／eM Client・Mailbird は有料寄り。
  - Mac には Spark・Mimestream 等があるが、**Windows は“美しさ・心地よさ”軸が手薄**。

---

## 3. 差別化（防御力のある3点＋補助）

1. **所有欲・アンビエント** — 全面ビジュアルのホーム＝置き時計ウィジェット化。常駐したくなる感情的差別化。
2. **AIからのプライバシー** — 保護領域（伏字＋暗号）、AIには伏字、ローカル処理。時流に合致し、前面に出すクライアントは少ない。
3. **軽さ** — Tauri で ~10MB。Electron 肥大の逆。体感で分かる技術的差別化。

**補助（使うほど効く）**: 独自スレッド再構築（引用解析・自動分割・再件名）。一言では売りにくいが定着の決め手。

> 看板は **①所有欲 ②反AIプライバシー**、裏打ちが **③軽さ**。

---

## 4. TSG One エコシステム / ビジネスモデル

- **フリーウェア**。TSG One アプリ群の一員（Primadoc・Doculator 等とデザイン言語・AI基盤を共有）。
- **AIトークンを配れる** = ユーザーが自前APIキー無しでAI機能を使える ＝ 導入摩擦が激減（個人開発では通常できない強み）。
- 単体ではなく「**Tesaguri/TSG の作業スイート**」という一貫した物語で価値を出す。

### ライセンス・公開方針
- **アプリ本体のコードは非公開（クローズド）／フリーウェア**。
- **セキュリティ提案（保護領域の相互運用仕様）だけを公開**し、他クライアントの実装を促す。
- 公開部分は**切り分けて** `spec/`（ベンダー中立・オープン）に置く。アプリ内部設計は `docs/`（非公開）。詳細: [../spec/README.md](../spec/README.md)。

### テレメトリ／プライバシー姿勢（看板と一致）
- **計測は最小・透明・オプトイン**。**メール本文等のコンテンツは送らない**。
- クラッシュレポートもオプトイン。スパム共有も“シグナルのみ・オプトイン”（[SPAM.md](SPAM.md)）。
- 「勝手にAIにもトラッカーにも渡さない」を製品全体の原則にする（[MAIL_SECURITY.md](MAIL_SECURITY.md)）。

## 5. 認証・設定方針（重要）

- **基本のメール利用に OAuth は不要。** 普通のメールクライアントと**同じ設定レベル**（IMAP/SMTP のホスト・ポート・ユーザー・パスワード／アプリパスワード）。
- **OAuth は AI・TSG One アカウント連携を使うときのみ**（AIを使わない人には不要）。
- ※注記（事実）: Gmail / Outlook.com は**サーバー側が**アプリパスワード（2FA前提）または OAuth を要求する場合がある。これはプロバイダ事情であり、本アプリの基本方針（手動設定で完結）は変えない。汎用 IMAP/SMTP はそのまま手動設定で動く。

---

## 6. 尖った MVP（最初に完成させる線引き）

**MVP = “1アカウントが鉄壁に動く、美しくて軽いWindowsメール”＋尖った2フック。**

含む:
- Windows 単体 / 1〜数アカウント（手動 IMAP/SMTP 設定）
- 受信・送信・チャット形式表示・**引用剥がし**
- **美しいホーム＋ウィジェット化**（背景画像の取り込み・時間/日替わり）
- **保護領域 Lv1**（ヘッダ方式＋アプリ共有鍵、伏字トグル・送信前確認）
- 検索（FTS5）・基本フィルタ（ブックマーク／要再確認）
- AI は**オプトイン**（TSG One トークン or ローカル）

後段（コア安定後）:
- 住所録 → カレンダー → フィルタ拡充 → スレッド再構築の精緻化
- **SNS 統合**（メッセージハブ）
- **モバイル（Expo）**
- 保護領域 Lv2（本人鍵）・OAuth プロバイダ対応

## 7. やらない / 今はやらないこと（スコープ規律）

- 重厚な管理機能・エンタープライズ運用（当面）。
- すべてのSNS網羅（不可なものは正直に穴として残す）。
- 「地味な信頼性」を後回しにした派手機能の先行（順序を守る）。

---

## 8. 成功の定義

- **第一**: 作者が毎日触って“気持ちいい・所有したい”と思える（自分用として完成）。
- **第二**: 同じ感性の Windows ユーザーのニッチに、看板①②で自然に刺さる。
- マスマーケット制覇は目標にしない（メールクライアントは激戦・粘着市場）。**忠実なニッチ＋エコシステム**で十分。

---

## 9. 心得（迷ったら）

- **1アカウントの鉄壁さ＞派手機能**。IMAP/SMTP の各社差・同期・検索の“地味な8割”が本体。
- **尖ったMVPを崩さない**。広げたくなったら本書の §6 に戻る。
- **看板の一文（§1）を通せるか**で機能採否を判断する。
