# 同期範囲・保持期間（取得する期間の選択）

**ステータス:** 計画（実装未着手）
**結論:** **すべてを無条件に取り込むのではなく、取得する期間をユーザーが選べる**（アカウント単位）。本文・添付は段階的に取得して軽快さと容量を両立する。

関連: [DEVELOPMENT_PLAN.md](DEVELOPMENT_PLAN.md)（Phase 4）/ [CROSS_PLATFORM.md](CROSS_PLATFORM.md)（独立 IMAP 同期）/ [DATABASE_SCHEMA.md](DATABASE_SCHEMA.md) / [DATA_STORAGE.md](DATA_STORAGE.md)

---

## 1. 同期ウィンドウ（ユーザー選択）

アカウントごとに「どこまで遡って取り込むか」を選べる。

- 選択肢（例）: **過去 1ヶ月 / 3ヶ月 / 6ヶ月 / 1年 / 2年 / 全期間**
- **既定: 6ヶ月**（軽快さと実用性のバランス。`全期間` はオプトイン）
- 端末ごとに異なる設定でよい（[CROSS_PLATFORM.md](CROSS_PLATFORM.md) の独立 IMAP 同期と整合。各端末が自分のウィンドウで同期）。
- 変更時: ウィンドウを広げたら不足分を追加取得、狭めたら範囲外をローカルから整理（下記 保持ポリシー）。

---

## 2. 段階的取得（メタデータ → 本文 → 添付）

全文を一度に落とさず、必要なものから取得して軽量・高速に保つ。

| レイヤ | 取得方針 |
|---|---|
| **ヘッダ/メタデータ** | 同期ウィンドウ内は取得。スレッド再構築・フィルタ・一覧の基盤（軽い） |
| **本文（clean_body）** | ウィンドウ内は取得（既定）。または開いた時にオンデマンド取得も選択可 |
| **添付ファイル** | **遅延取得**（開いた時／明示ダウンロード時）。サイズ上限・自動DL条件は設定可 |

> これにより「メタデータは広く・本文/添付は必要分だけ」を実現し、初回同期と容量を抑える。

---

## 3. 保持ポリシー（retention）

ローカルにどこまで残すかを選べる。

- **`window`（既定）**: 同期ウィンドウ内のみ保持。範囲外はローカルから整理。
- **`metadata_only`**: 範囲外も**メタデータ（ヘッダ・索引）は保持**し、本文/添付だけ削除して容量節約（検索ヒットは残る）。
- **`keep_all`**: 取得したものは消さない（容量優先でないユーザー向け）。
- 整理した本文/添付は**必要時に IMAP サーバーから再取得**（サーバーが源 → ローカルは可変キャッシュ）。

---

## 4. ウィンドウ外の検索

- ローカルは FTS5 で高速検索（ウィンドウ内 or メタデータ保持分）。
- **ウィンドウ外を探す時はサーバー検索（IMAP SEARCH）をオンラインで実行**するオプションを用意。ヒットしたものだけ取り込む。

---

## 5. 同期の流れ

```
初回: 選択ウィンドウ（例 6ヶ月）の範囲だけ取得 → 軽い初回同期
以降: 差分同期（UIDVALIDITY / UIDNEXT 管理）で新着を取得
随時: 本文/添付はオンデマンド、古いものは保持ポリシーに従い整理
```

---

## 6. データモデル（accounts に設定列を追加）

詳細は [DATABASE_SCHEMA.md](DATABASE_SCHEMA.md)。

- `accounts.sync_window`           — `'1m'|'3m'|'6m'|'1y'|'2y'|'all'`（既定 `'6m'`）
- `accounts.body_fetch`            — `'window'|'on_demand'`（既定 `'window'`）
- `accounts.attachment_fetch`      — `'on_demand'|'auto'`（既定 `'on_demand'`）
- `accounts.retention`             — `'window'|'metadata_only'|'keep_all'`（既定 `'window'`）

---

## 7. Tauri コマンド（抜粋）

| コマンド | 用途 |
|---|---|
| `account_set_sync_window` | 同期ウィンドウ変更（不足分の追加取得をトリガ） |
| `account_set_retention` | 保持ポリシー変更 |
| `message_fetch_body` / `attachment_download` | 本文・添付のオンデマンド取得 |
| `server_search` | ウィンドウ外をサーバー側（IMAP SEARCH）で検索 |
